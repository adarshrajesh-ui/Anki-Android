## Shared engine — what's proven vs. next step (honest)

- **Proven:** AnkiDroid runs on the **Anki Rust backend** — the same Rust engine family as the
  desktop fork. Import, scheduling (FSRS), answer logging and DB integrity are all executed by the
  Rust `Backend`/mediasrv (`net.ankiweb.rsdroid.Backend`), consumed as the published dependency
  `io.github.david-allison:anki-android-backend:0.1.64-anki25.09.2` (upstream Anki 25.09.2). No
  scheduler logic was reimplemented in Kotlin — AnkiDroid's `Scheduler.kt` is a thin pass-through
  over the generated backend.
- **Not yet wired (next step):** the *fork's own* custom RPCs. The desktop fork's custom engine work
  lives on branch **`cfa/exam-queue-mvp`** (desktop repo `~/AlphaWeek2/ankiCFA`): the read-only
  **`SchedulerService.BuildExamQueue`** RPC (`proto/anki/scheduler.proto` + `rslib/src/scheduler/service/mod.rs`
  — reorders due cards by topic_weight × (1−retrievability) × deadline_urgency, writes nothing) plus an
  honest per-topic memory score (`pylib/anki/cfa.py`). (A separate earlier `mastery` query,
  `rslib/src/stats/mastery.rs`, sits on the `gnhf/context-discipline-i-ef2200` branch of the other
  `~/wed/ankiCFA` copy.) Neither is in the *stock* backend AnkiDroid consumes here. To expose fork-specific RPCs on Android:
  clone `Anki-Android-Backend`, point its `anki` submodule at this fork's `proto/`+`rslib`,
  cross-compile the JNI `.so` (NDK + Rust android targets) to produce `rsdroid-release.aar`, and set
  `local_backend=true` in `local.properties` (AnkiDroid already supports this substitution —
  `AnkiDroid/build.gradle` / `buildSrc/.../BackendDependencies.kt`). The generated `Backend` then
  gains `col.backend.buildExamQueue(...)`, surfaced via a one-line `Scheduler.kt`-style pass-through.
  This is a multi-hour cross-compile and was deliberately deferred (Wednesday de-risk gate is a real
  review on the shared engine; sync/fork-RPC are Fri).

## Reproduce

```bash
# 0. build + APK (build gate)
bash scripts/verify_mobile.sh

# 1. install emulator + system image (once)
source scripts/adb_env.sh
yes | "$SDKMANAGER" "emulator" "system-images;android-34;google_apis;arm64-v8a"
echo no | "$AVDMANAGER" create avd -n ankidroid_cfa -k "system-images;android-34;google_apis;arm64-v8a" --device pixel_6

# 2. boot headless
"$EMULATOR" -avd ankidroid_cfa -no-window -no-audio -no-boot-anim -no-snapshot -gpu swiftshader_indirect &
"$ADB" wait-for-device

# 3. install + import + record + kill test
"$ADB" install -r -g "$APK"
python3 scripts/make_cfa_deck.py            # -> cfa_level2.apkg  (then import via app: overflow -> Import -> Deck package -> pick file)
brew install scrcpy                          # host-side recorder (on-device screenrecord won't finalize under input load)
bash scripts/set_due_full.sh && bash scripts/review_scrcpy.sh   # records docs/mobile-proof/cfa_review_session.mp4
bash scripts/set_due_full.sh && bash scripts/kill_test.sh       # 3x kill-mid-review + integrity
```
