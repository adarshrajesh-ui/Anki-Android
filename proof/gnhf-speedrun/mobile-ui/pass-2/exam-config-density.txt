Phase B — Pass 2 (mobile, harsher) — M4-2: Exam Config sparse / no rationale
============================================================================

Screen: Exam configuration (CfaExamConfigActivity)
Device: emulator-5554 (real running fullDebug build), adb screencap.

DEFECT (Pass-1 carried MINOR M4-2)
----------------------------------
The Exam configuration screen was a bare title + date field + Pick date + Save,
with a large empty lower half and NO explanation of why setting an exam date
matters, and NO feedback once a date was chosen. On a premium paid exam-prep
product a config screen that just says "No exam date set" over a wall of empty
space reads as unfinished — the user has no reason to engage and no confirmation
of what they picked.

FIX (this pass)
---------------
Presentation-only (no change to the persisted `cfa_exam_config` col.conf shape,
which stays byte-identical to the desktop reference so the synced key still
drives both apps):

1. Context line under the title (cfa_exam_config_why):
   "Set your exam date so ankiCFA can weight study by points-at-stake and show a
    live countdown on the Exam Readiness screen."
   Renders in calm cfa_muted grey — tells the user why the field matters.

2. Live countdown preview under the date field (cfa_config_countdown), shown the
   moment a date is set and on re-open:
     future  -> "N days to the exam"   (plurals: 1 day / N days)
     today   -> "Your exam is today — good luck."
     past    -> "This exam date has already passed."
   Rendered in the warm cfa_accent (the single warm accent on CFA screens),
   hidden entirely when no date is set (visibility=gone).

   Backed by a pure, testable helper CfaExamConfig.daysUntil(date, today) that
   returns whole days (positive future / 0 today / negative past) or null when
   the date is unset/blank/unparseable — no device clock or resources needed to
   unit-test the count.
