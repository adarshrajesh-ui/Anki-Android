GOAL
----
Review the SAME card on BOTH devices while offline, sync, and show the
documented conflict rule picks a clear winner — device-observable, no fabrication.

THE RULE (documented)
---------------------
"More-recent review wins." The card's final scheduling STATE reflects whichever
review happened later in wall-clock time (mirrors Anki's own last-writer-wins:
higher modification time is kept). BOTH reviews still persist in the revlog as
distinct millisecond-id rows, so no review is lost and none is double-counted;
only the current card state is decided by the rule.
Reference: desktop pylib/anki/cfa_sync.py::resolve_review_conflict + the
revlog is append-only and keyed by unique ms ids.

SCENARIO (a genuine offline divergence from a shared base)
----------------------------------------------------------
Both devices start synced to the server (BASE, revlog=28). Then, independently:

1. PHONE goes OFFLINE (airplane mode; `ping 10.0.2.2` => Network unreachable).
   In the AnkiDroid reviewer it answers card X = 1783137755188
   ("What defines a hedge fund as an investment vehicle?") = AGAIN (ease 1).
   -> phone card X: reps 0->1, queue=1 (learning), ivl~60s.
   -> phone revlog row p_id = 1783212583931 (ease=1). Phone revlog 28->29.
   -> screenshots 01/02 show the airplane icon (offline) + the 4 ease buttons.

2. DESKTOP reviews the SAME card X from the BASE (the phone's Again is NOT on
   the server yet — phone is offline). It answers card X = EASY (ease 4) at a
   LATER wall-clock time (desktop revlog id 1783212648010 > phone 1783212583931).
   -> desktop card X: reps 0->1, queue=2 (review), ivl=4d.
   -> desktop uploads. Server now has the desktop review.
   DESKTOP_CONFLICT_MARKER (from cfa_desktop_conflict_review.py):
     card_id=1783137755188  reps_before=0 reps_after=1  desktop_answer=Easy(4)
     desktop_revlog_id=1783212648010  desktop_card_queue=2 type=2 ivl=4
     revlog_base=28 (phone's offline Again absent, as expected)

3. PHONE comes back ONLINE and syncs (logcat: "sending done=true ... revlog=1"
   = phone uploaded its offline Again; then the merge pulls the desktop review).
