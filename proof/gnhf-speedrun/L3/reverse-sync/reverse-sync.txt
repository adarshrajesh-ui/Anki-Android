ROOT CAUSE OF THE HANDOFF "DIVERGENCE" (honest finding)
The HANDOFF observed the on-disk collection.anki2 showing only ~3 revlog rows
"that does not reflect the synced state". That was a MEASUREMENT ARTIFACT, not
a data-loss/reset bug: reading collection.anki2 while the app is LIVE (or via
`adb pull` of the main file + a separately-pulled -wal from a running process)
shows only the checkpointed pages — the synced reviews are durably held in the
un-checkpointed WAL. After a clean cold-stop the WAL either checkpoints into the
main file (round 1: main file = 26 revlog, WAL 0 bytes) or remains a consistent
WAL that SQLite applies on next open (round 2: 28 revlog). Either way the synced
state PERSISTS and the phone UI reflects it. No divergence, no reset, no
corruption. The IntentHandler "routes to Cfa* instead of DeckPicker" concern is
also stale: this branch's IntentHandler has no CFA routing; launch goes straight
to DeckPicker.
